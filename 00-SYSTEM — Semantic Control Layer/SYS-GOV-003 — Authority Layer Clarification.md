# @SYS-GOV-003 — Authority Layer Clarification

> Purpose: Clarify authority-layer interpretation boundaries without changing framework structure, ontology hierarchy, or doctrine.

## Authority Layer Definitions

- **SYS** = semantic doctrine (primary conceptual governance source).
- **CAN** = canonical enforcement layer derived from SYS.
- **ARC** = operational application of doctrine inside framework architecture.
- **RULE** = agent-facing operational guidance and behavioral interpretation.
- **LEGACY** = historical reference only; non-authoritative for active governance decisions.

## Naming Authority Hierarchy

1. **Primary Naming Authority:** `@SYS-STD-002`.
2. **Operational Inheritance:** `@ARC-STD-007` inherits operationally from `@SYS-STD-002`.
3. **Agent Interpretation Layer:** `RULE-005` is interpretation guidance for agents and does not override SYS/CAN authority.
4. **Legacy Constraint:** legacy files (including LEGACY-prefixed documents) are non-authoritative and must not be used as active naming governance sources.

## Non-Destructive Boundary Rule

This clarification does not merge layers and does not centralize governance into a single artifact.
SYS remains doctrinal authority, CAN remains derivative enforcement, and ARC/RULE remain application layers.
