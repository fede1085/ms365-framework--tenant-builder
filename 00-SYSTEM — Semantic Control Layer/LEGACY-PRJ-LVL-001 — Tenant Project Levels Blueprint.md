# PRJ-LVL-001 — Tenant Project Levels Blueprint

## Objective

Define a structured progression model to build Microsoft 365 tenants.

---

## Levels Overview

### Level 0 — Raw Tenant

- Tenant created
- Domain configured
- Break-glass admin

---

### Level 1 — Identity Baseline

- Users created
- Licenses assigned
- Naming applied
- Admin separation

---

### Level 2 — Operational Structure

- Departments defined
- Groups / Teams created
- Shared mailboxes created
- Membership assigned

---

### Level 3 — Governance & Control

- Owners defined
- Permission model applied
- Naming standardized
- Lifecycle defined

---

### Level 4 — Automation & Deployment

- Blueprint finalized (BLP)
- Matrix generated (MTX)
- Scripts prepared (AUT)

```text
Level 4 = where automation starts
MTX = execution layer
```

---

### Level 5 — Security & Maturity

- MFA enforced
- Conditional Access
- Compliance / retention
- Monitoring & reporting

---

## Key Mapping

```text
Level 3 → ARCH + BLP
Level 4 → MTX + AUT
```

---

## Usage

```text
0 → 1 → 2 → 3 → 4 → 5
```
