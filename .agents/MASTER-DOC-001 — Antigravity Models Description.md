# Cómo se sienten en la práctica

## 🔥 Gemini 3.1 Pro (High)

Tu “cerebro pesado”.

Muy bueno para:

* frameworks
* arquitectura
* reasoning
* analizar muchos archivos
* contexto enorme
* multimodal

Se siente:

> técnico, estructurado, analítico.

A veces:

* menos natural
* menos “humano”
* puede sobreexplicar.

---

## ⚡ Gemini 3.1 Pro (Low)

Es el mismo “tipo de cerebro” pero pensando menos.

Imagínalo así:

* High = ingeniero senior concentrado
* Low = mismo ingeniero pero apurado

Sirve para:

* ahorrar cuota
* tareas medias
* continuidad de contexto

---

## 🚀 Gemini 3 Flash

Velocidad.

MUY rápido.
Barato computacionalmente.

Ideal para:

* brainstorming
* transformar texto
* iterar rápido
* clasificar cosas
* tareas repetitivas

No tan bueno para:

* reasoning profundo
* arquitectura compleja
* debugging difícil.

---

# Claude Sonnet 4.6 (Thinking)

Probablemente el mejor “balance”.

Muchísima gente lo usa como daily driver.

Muy bueno para:

* código
* refactors
* UX
* explicaciones
* agentes
* workflows reales

Se siente:

> inteligente + práctico + natural.

Generalmente:

* más consistente que Gemini
* menos “robot”.

Muchos developers prefieren Sonnet para trabajar todos los días. ([Tom's Guide][1])

---

# 🔥 Claude Opus 4.6 (Thinking)

El monstruo.

Más lento.
Más caro computacionalmente.
Pero brutal.

Ideal para:

* problemas MUY difíciles
* reasoning largo
* planificación compleja
* agentes autónomos
* código enorme

Se siente:

> “senior architect”.

Mucha gente dice que es el mejor modelo actual para coding complejo y agentes largos. ([IT Pro][2])

---

# GPT-OSS 120B

Importante:

“OSS” = Open Source / Open Weight.

Generalmente:

* menos alineado/pulido
* más “raw”
* puede sorprender
* a veces inconsistente

Bueno para:

* experimentar
* tasks técnicas
* velocidad local/open ecosystem

Pero normalmente:

> debajo de Gemini Pro / Claude Sonnet / Opus.
